Flow Logger

The flow logger is used when logging surrounds an execution flow.

You can access the flow logger from Blindsight using logger.flow:

import com.tersesystems.blindsight._

val logger = LoggerFactory.getLogger
val flowLogger = logger.flow

Usage

The flow logger takes a block of execution, and returns the result transparently, according to the log level. It takes an implicit flow mapping, which determines the behavior of the logging flow. 

implicit val flowMapping: FlowMapping[B] = ...

def flowMethod(arg1: Int, arg2: Int): Int = logger.flow.trace {
  arg1 + arg2
}

The result must have a type class instance of ToArguments, so that it can be considered in logging. For example, if the return type is Person, then you must have a ToArguments[Person] in scope:

def personFlowMethod(arg1: Int, arg2: Int): Person = logger.flow.trace {
  Person(name = "Will", age = arg1 + arg2)
}

case class Person(name: String, age: Int)

object Person {
  implicit val personToArguments: ToArguments[Person] = ToArguments { person =>
    Arguments(kv("person", person.name), kv("age", person.age))
  }
}

Flow Mapping

Implementing a flow mapping behavior comes down to creating Markers and Statements for entry, exit, and exceptions.

case class Source(line: Line, file: File, enclosing: Enclosing, args: Args)

trait Mapping[B] {
  def entryMarkers(source: Source): Markers    = Markers.empty
  def exitMarkers(source: Source): Markers     = Markers.empty
  def throwingMarkers(source: Source): Markers = Markers.empty

  def createEntryStatement(source: Source): Statement

  def createExitStatement(
    resultValue: B,
    source: Source
  ): Statement

  def createThrowingStatement(
    exc: Throwable,
    source: Source
  ): Statement
}

For example, you can replicate the behavior of the SLF4J XLogger methods transparently, by passing in the appropriate markers and using the logstash implicits to convert source code arguments.

import com.tersesystems.blindsight.logstash.Implicits._
import net.logstash.logback.argument.StructuredArguments._

class XLoggerFlowMapping[B: ToArguments] extends FlowLoggerMethod.Mapping[B] {
  override def entryMarkers(source: Source): Markers    = XLoggerFlowMapping.entryMarkers
  override def exitMarkers(source: Source): Markers     = XLoggerFlowMapping.exitMarkers
  override def throwingMarkers(source: Source): Markers = XLoggerFlowMapping.throwingMarkers

  override def createEntryStatement(source: Source): Statement = {
    import com.tersesystems.blindsight.logstash.ToArgumentsImplicits._

    Statement()
      .withMarkers(entryMarkers(source))
      .withMessage(s"${source.enclosing.value} entry {}")
      .withArguments(source.args)
  }

  override def createThrowingStatement(throwable: Throwable, source: Source): Statement = {
    Statement()
      .withThrowable(throwable)
      .withMarkers(throwingMarkers(source))
      .withMessage(s"${source.enclosing.value} exception")
  }

  override def createExitStatement(resultValue: B, source: Source): Statement = {
    Statement()
      .withMarkers(exitMarkers(source))
      .withMessage(s"${source.enclosing.value} exit with result {}")
      .withArguments(resultValue)
  }
}
